## 貢献ガイドライン

1. [スタイルガイド](./style-guide.md)に従う
2. ドキュメントをモジュール化し、焦点を絞る
3. 図、例、用語集リンクを含める
4. 現在形と能動態を使用する
5. 頭字語を定義し、関連コンテンツを参照する

## ドキュメント標準

### ファイル命名

* ハイフンを使った小文字を使用: `cognitive.md`、`overview-v2.md`
* トピック固有のドキュメントは適切なフォルダに配置

### 構造テンプレート

1. 明確なタイトルと目的
2. 長いファイルの目次
3. 例を含む論理的なセクション分け
4. 関連概念間のクロスリンク

### スタイル規範

* 能動態、現在形
* 一貫した技術用語
* 初出時の頭字語定義
